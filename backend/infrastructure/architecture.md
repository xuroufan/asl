## 组件通信关系

### 请求链路 (下单流程示例)

```
用户点击 [下单] → WebSocket/HTTP → Gateway → Order Service
                                                      │
                                                      ├─▶ Nacos (服务发现)
                                                      ├─▶ Redis (幂等性检查)
                                                      ├─▶ MySQL (订单持久化)
                                                      ├─▶ RocketMQ (发送 order-created)
                                                      │       └─▶ Matching Engine (消费)
                                                      │              ├─▶ 成交 → RocketMQ → Order更新状态
                                                      │              ├─▶ Redis → Position缓存更新
                                                      │              └─▶ WebSocket → 推送至终端
                                                      ├─▶ Seata TCC (分布式事务)
                                                      │       ├─▶ Fund Service (冻结保证金)
                                                      │       └─▶ Risk Service (验仓)
                                                      └─▶ SkyWalking (全链路追踪)
```

## 数据流向

```
行情数据: 交易所 → Market Service → Kafka → Matching (计算) → Redis (缓存)
                                     └─→ WebSocket → 用户终端

订单数据: 用户 → Gateway → Order Service → RocketMQ → Matching Engine
              ↓                                              ↓
          Redis (幂等)                                  RocketMQ ← 成交事件
              ↓                                              ↓
          MySQL (持久化)                              Fund/Position 更新
```
