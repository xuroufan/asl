# 期货交易平台 — SLA / SLO / SLI 定义

## 1. 服务等级目标 (SLO)

| 指标 | 目标 | 测量窗口 |
|------|------|----------|
| 核心交易API可用性 | 99.95% | 月度 |
| 网关API可用性 | 99.99% | 月度 |
| 行情推送延迟(P99) | <500ms | 日 |
| 订单撮合延迟(P99) | <50ms | 日 |
| 订单接收成功率 | >99.9% | 日 |
| API响应时间(P99) | <2s | 日 |
| 数据库查询延迟(P99) | <200ms | 日 |

## 2. 服务等级指标 (SLI)

### 可用性
```
SLI_availability = (总请求数 - 5xx错误数) / 总请求数
```

### 延迟
```
SLI_latency_p99 = http_server_requests_seconds_sum / http_server_requests_seconds_count
```

### 吞吐量
```
SLI_throughput = rate(http_server_requests_seconds_count[5m])
```

## 3. 服务等级协议 (SLA)

| 可用性 | 响应时间 | 补偿 |
|--------|----------|------|
| <99.9% | 4小时 | 当月费用10% |
| <99.0% | 2小时 | 当月费用25% |
| <95.0% | 30分钟 | 当月费用100% |

## 4. 错误预算

```
每月错误预算 = (1 - SLO) × 总分钟数
99.95% = 0.05% × 43200分钟 = 21.6分钟/月
```

### 消费速率
- 每次P1告警 = 消耗5分钟预算
- 每次P2告警 = 消耗2分钟预算
- 达到80% → 冻结新功能上线
- 达到100% → 启动紧急恢复流程

## 5. 告警优先级

| 级别 | 响应时间 | 例 |
|------|----------|-----|
| P1-Critical | 15分钟 | 服务宕机、订单失败 |
| P2-High | 30分钟 | 高延迟、高错误率 |
| P3-Medium | 2小时 | 资源使用率高 |
| P4-Low | 24小时 | 性能降级、非关键告警 |

## 6. 值班响应流程

```
P1告警 → 15分钟内响应 → 接入Slack频道
         → 30分钟内定位 → 启动War Room
         → 1小时内恢复 → RCA在24小时内完成

P2告警 → 30分钟内响应 → 接入Slack频道
         → 2小时内定位 → 修复
         → RCA在48小时内完成

日常维护 → 按迭代计划 → 变更窗口内完成
```
